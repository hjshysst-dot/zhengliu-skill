# 胖叔蒸馏法 - 快速启动

## 一句话介绍

把一本书/一个人的知识，蒸馏成一组可执行、可测试、可进化的技能包。

## 核心流程

```
阶段0: 知识理解 → 源头质量评估
阶段1: 协调提取 → 5并行 + Coordinator去重
阶段1.5: 四重验证 → V1跨域/V2预测力/V3独特性/V4自洽
阶段2: RIA++E构造 → 触发条件量化
阶段3: 达尔文进化 → 边界探测 + 评分rubric
阶段4: 自动化维护 → 变更传播 + 版本锁定
```

## 快速调用

```
用户：帮我蒸馏《穷查理宝典》
→ 胖叔蒸馏法自动启动
→ 阶段0：确认来源质量
→ 阶段1-3：自动执行
→ 阶段4：通知用户确认
```

## 三步走示例

### Step 1: 确认输入
```
胖叔：需要蒸馏什么？
- 书（PDF/EPUB/TXT）
- 人的资料（访谈/著作/transcript）
- 用来解决什么问题？
```

### Step 2: 自动执行
```
→ 源头质量评估
→ 协调提取 + 去重
→ 四重验证筛选
→ RIA++E构造
→ 边界探测测试
→ 达尔文进化
```

### Step 3: 用户确认
```
产出报告：
- 总共提炼出X个skill
- 通过率：Y%（通过了四重验证）
- 达尔文进化：Z轮优化
- 建议立即可用的：N个
- 需要人工审核的：M个
```

## 与Cangjie的核心区别

| Cangjie | 胖叔蒸馏法 |
|---------|----------|
| 5个并行extractor各自为战 | Coordinator协调层统一管理 |
| 三重验证 | 四重验证（+自我一致性） |
| 诱饵测试（鸡生蛋） | 边界探测（可操作） |
| V3过于严格 | 应用密度豁免条件 |
| 无源头质量检测 | 源头质量前置评估 |
| 外部维护成本高 | 内置自动化维护 |
| INDEX手动维护 | 变更触发自动重建 |
| 无原子化标准 | 触发条件量化标准 |

## 适用场景

- 想把一本书的方法论做成可调用的工具
- 想把某个牛人的思维方式提炼成skill
- 想让现有skill持续进化而不是衰减
- 想建立团队知识传承体系

## 不适用场景

- 简单书摘/读后感（用普通总结工具）
- 角色扮演/人设模仿（用nuwa-skill）
- 单次咨询不需固化（直接问就行）
